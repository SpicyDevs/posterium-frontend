Builder UI — Concept Board

Single-page rough outline with 5 desktop layout ideas (simple + advanced modes) and 10 phone-native UI ideas.
Route target: /experimental (file: public/experimental/index.html).

Desktop Visual Ideas (for Discord sharing)

1) Studio Split Canvas (Simple)

Goal: 70% canvas + 30% right settings rail. Fast and focused beginner flow.

2) Dockable Pro Workspace (Advanced)

Goal: Multi-panel desktop app feel with dockable layers, inspector, timeline.

3) Command Palette First (Simple→Advanced)

Goal: Keyboard-first productivity with slash commands and contextual actions.

4) Progressive Steps + Expert Toggle

Goal: Wizard-like simple mode, same screen unlocks expert controls in place.

5) Data Grid + Live Preview (Advanced)

Goal: Spreadsheet/panel for power users with synchronized visual preview.

Phone-Native UI Ideas

1) Bottom Tabs + Floating Create

Native 5-tab structure, thumb-reachable FAB for primary action.

2) Sheet-Based Editing

Edit controls open as bottom sheet while canvas stays fully visible.

3) Gesture-Driven Timeline

Horizontal scrub + haptics; pinch to zoom timeline density.

4) Card Stack Builder

Step cards (content/style/publish) with swipe navigation and progress chip.

5) One-Hand Quick Actions

Arc menu near thumb zone for duplicate, resize, align, and undo.

6) Native iOS/Android Theming

Platform-specific components: segmented controls vs material chips.

7) Contextual Toolbelt

Tools appear only when object selected, reducing noise for novices.

8) Voice + Prompt Assist

Microphone/prompt bar to create edits conversationally on mobile.

9) Multi-Device Handoff

Persistent session card: continue on desktop without losing position.

10) Accessibility-First Mobile

Large hit targets, high contrast modes, and readable dynamic type scale.
